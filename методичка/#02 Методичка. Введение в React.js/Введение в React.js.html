В 2013 Facebook представила библиотеку react.js. React.js помогает создавать масштабируемые веб-приложения на JavaScript. В основе реакта лежат простые концепции, вместе они создают новый удобный подход, декларативного описания пользовательского интерфейса.
1. Рендер react в DOM
Чтобы добавить код на реакт в DOM, нужно воспользоваться простой функцией ReactDOM.render.

Если посмотреть файл src/index.js в новом проекте create-react-app, можно увидеть команду: 
 ReactDOM.render(<App />, document.getElementById('root'))
Таким образом происходит добавление всего приложения на реакте в дерево DOM с id root. После этого приложение на реакт начинает отвечать за все содержимое в этом теге. Если там было что-то, реакт очистит содержимое и начнет заполнять его теми тегами, которые вы опишете в своем приложении.

Содержимое этого тега будет представлять из себя ваше приложение, оно может быть как spa, с описанием роутинга, так и простым виджетом на вашем сайте, умной формой или калькулятором цен.

С помощью ReactDOM.render можно подключить несколько приложений на вашей странице, и каждое из них будет работать в своем изолированном теге, со своей логикой, без конфликтов с другими приложениями на странице.
2. Концепции react.js
Легче всего понять реакт, изучив базовые концепции, лежащие в его основе.
Их всего 3: virtual DOM, компонентная разработка и jsx.
1
Virtual DOM
Virtual DOM — это модель DOM, описываемая с помощью простых объектов js.

Одной из причин внедрения virtual dom в react стало то, что работа с обычном DOM является достаточно медленной. Если вы попробуете сдвинуть тысячи div на 5 пикселей влево, вы увидите провисание fps в браузере.

Второй причиной является то, как реакт работает с изменениями ui. В классической работе jquery, к началу работы скрипта мы уже имеем наполненное DOM дерево. Если мы хотим как-то повлиять на внешний вид, мы или добавляем, или удаляем, или меняем теги в дереве.

В случае реакта, мы описываем внешний вид компоненты (кнопка, отдельное меню или целая страница) с помощью jsx-тегов. Когда такая часть страницы должна быть выведена на экран, реакт строит virtual dom. После построения virtual dom компоненты, реакт подключает ее к общему дереву virtual dom всей страницы, и сравнивает то, что изменилось.

Если мы первый раз выводим компоненту на экран, реакт с помощью простых операций document.createElement, document.createTextNode, и тд. добавляет компонент в реальный DOM. Если же мы изменили данные, и нам нужно показать компонент с новыми измененными данными, реакт опять строит virtual dom компоненты с новыми данными и проводит сравнение версий virtual dom, прежней с текущей.

Снова происходит процесс выявления изменений, и в данном случае к dom применяются только те изменения, которые были выявлены при изменении внешнего вида, который поменялся при изменении данных.
Virtual DOM создает абстракцию над созданием и обновлением тегов в реальном DOM, с реактом вся работа сводится к работе с jsx и логикой, описанной в компонентах.
2
Компонентная разработка
Компонентная разработка — это способ композиции кода в мире реакт. Компонентой называется класс или функция, которая описывает элемент или группу элементов, связанных визуально или логически.
На данный момент есть 3 основных способа создания компоненты:
1. Расширить класс Component

class HelloWorld extends React.Component {
 render() {
   return <div>Hello!</div>;
 }
}

2. Расширить класс PureComponent
class HelloWorld extends React.PureComponent {
 render() {
   return <div>Hello!</div>;
 }
}

3. Написать функцию возвращающую jsx

function HelloWorld () {
  return <div>Hello!</div>
}


//Также, вы можете увидеть старый способ, который уже не пользуется популярностью

React.createClass({
 render: function() {
   return <div>Hello!</div>;
 }
});
Компоненты содержат вёрстку в jsx, логику работы компоненты и, как правило, импортируют стили самой компоненты, это может быть как css, так и одна из реализаций css-in-js.
Компоненты могут содержать в себе другие компоненты. Компонент списка комментариев будет возвращать контейнер, например <ul></ul>, в котором будут помещены компоненты комментариев, которые в свою очередь могут вернуть или просто jsx, или другие компоненты.

Компоненты получают данные через props. Props — это атрибуты тега компоненты, например: 
class Comment extends Component {
 render() {
   const {author, text} = props;
   return (
     <div>
       <h4>{author}</h4>
       <p>{text}</p>
     </div>
   );
 }
}

class CommentList extends Component {
 render() {
   const comments = [
     {text: 'hello!', author: 'Bill'},
     {text: 'How are you?', author: 'Mary'}
   ];
   return (
     <ul>
       {comments.map(comment => {
         return <Comment author={comment.author} text={comment.text} />;
       })}
     </ul>
   );
 }
}
Компонент CommentList возвращает список компонент комментария, передавая ему через атрибуты тэга данные. Эти данные компонент комментариев и получает через this.props.

В случае, когда мы используем функцию, а не класс, props передается аргументом функции: 
function Comment(props) {
 const {author, text} = this.props;
 return (
   <div>
     <h4>{author}</h4>
     <p>{text}</p>
   </div>
 );
}

//Или если совсем коротко:

const Comment = ({author, text}) => (
 <div>
   <h4>{author}</h4>
   <p>{text}</p>
 </div>
);
Компоненты можно и даже нужно использовать много раз, в случае комментария мы используем один и тот же компонент в списке комментариев. Можно упростить себе жизнь, создав на проект общий набор ui компонент, таких как кнопки, инпуты и тд, и использовать их везде в проекте, чтобы сохранить общий стиль проекта и не повторять оформление стиля.

Компоненты можно представить чистой функцией от props.
Чистая функция в программировании — это такая функция, которая вычисляет результат только в зависимости от переданных ей аргументов. Такая функция не использует внешние (глобальные) переменные, не изменяет свои аргументы и всегда возвращает один и тот же результат от одних и тех же аргументов.
Слово Pure в PureComponent именно это и означает: если props остались прежними, компонента не будет заново создавать свой virtual dom слепок, и реакт будет использовать прежний слепок.
3
JSX
JSX — это способ описать html-вёрстку в js. JSX не поддерживается браузерами, это надстройки над синтаксисом js. Чтобы jsx работал, нужен babel, и его правильная настройка.

После того, как babel обработает ваш код, на выходе получается функция React.createElement().
Тег <div/> в jsx превратиться в конструкцию React.createElement("div", null). JSX — это синтаксический сахар, который помогает описывать вёрстку в привычном для нас виде, но никто не запрещает писать React.createElement, правда этого никто не делает :-)

У JSX есть ограничения, если мы хотим присвоить класс тегу, то должны писать className, вместо class, так как class — зарезервированное слово в js для создания классов.

Для того чтобы вычислить что-нибудь прямо в jsx или использовать значение из переменной, используются фигурные скобки <p>{1 + 1}</p>, их можно использовать и для присваивания значений атрибутам тега
<input value={1000} />.

В фигурные скобки можно помещать значения и выражения, но нельзя помещать циклы или условия. Чтобы лучше понять это, нужно помнить, что весь jsx превращается в вызовы функций React.createElement, а значения атрибуты помещаются в аргументы этой функции.

Аргументами функции как раз и могут служить значения и выражения, но не циклы или условия — <input value={1000}/> => React.createElement("input", { value: 1000 }).

Можно использовать тернарный оператор, и это самый простой способ ветвления кода в jsx, например: 
<div>
{1 == 1
 ? <p>one equal one!</p>
 : <p>one not equal one!</p>
}
</div>
 
// Превращается в 

React.createElement(
 "div",
 null,
 1 == 1 ? React.createElement(
   "p",
   null,
   "one equal one!"
 ) : React.createElement(
   "p",
   null,
   "one not equal one!"
 )
)
Название компонент в jsx всегда должно начинаться с большой буквы, если у нас есть компонента P, в вёрстку попадет содержимое компоненты P, а если будет маленькая p, у нас в вёрстке появится параграф <p></p>.

И еще одно: если мы передадим компоненте атрибут без значения, то компонента получит этот атрибут со значением true.
3.Состояние компоненты
Помимо props, у компоненты есть свое состояние. Состояние — это простой объект. Состояние можно обновлять, и каждое обновление состояния вызывает новый render компоненты. Доступ к state происходит с помощью this.state. Состояние компоненты можно получить в любой функции компоненты. Изменить состояние можно с помощью функции this.setState. Также можно инициализировать состояние в конструкторе.
this.setState можно использовать двумя способами:
Передать объект — this.setState({a: 1})
Передать функцию — this.setState(state => ({...state, a: 1}))
В первом случае, если у нас есть состояние {a: 1, b: 2} и мы вызовем this.setState({a: 2}), то состояние станет {a: 2, b: 2}, а если состоянием будет объект {a: {b: 1, c: 2}} и вызовем this.setState({a: b 2}), у нас получится {a: {b: 2}}.

setState при передачи работает таким образом: заменяются ключи 1ой уровни вложенности на ключи state объекта. Если вы передаете ключём еще один объект, то он полностью заменить объект находящийся в state.

setState не изменяет состояние сразу же, react может задержать выполнение setState, чтобы обновить состояние сразу нескольких компонент одним проходом. Если вы посмотрите значение переменной this.state сразу после setState, то не увидете там изменений, которые вы передали. Если вам нужно иметь значение состояния сразу же после присваивания, вам нужно использовать второй аргумент setState: 
console.log(this.state); // {a: 1}
this.setState({a: 2}, () => {
 console.log(this.state); // {a: 2}
});
Если нужно сделать несколько изменений подряд, то лучше передать функцию, так как при вызове функции в нее передается актуальное состояние состояния.

У функциональных компонент, созданных лишь одной функцией, нет состояния и функций жизненного цикла.
4. Children
Компоненты становятся простыми тегами в мире реакта, а в каждый тег можно поместить потомков — <Parent><Children /></Parent>.

Для работы с детьми у компоненты есть свойство props.children. Реакт передаёт всех детей этой переменной, ребёнка можно разместить в том месте, где это удобно. Разместить только при условии или вовсе не работать с детьми. В таком случае, что бы вы не вложили в тег компонента, ничего не отразится в дереве DOM.

В случае с Parent, его код мог бы выглядеть так 
const Parent = props => (
 <div>
   <p>Hello! I am Parent!</p>
   {props.children}
 </div>
);
Любой ребенок, переданный в компонент Parent, будет размещён в нем после параграфа. Если же не передать ребенка, он просто не будет отражён у родителя.

Для работы со списками детей у реакта есть функция помощник React.Children.map, и если вы хотите обернуть каждого ребенка в дополнительный тег, она в этом поможет: 
const Parent = props => (
 <div>
   <p>Parent</p>
   {React.Children.map(props.children, child => <p>{child}</p>)}
 </div>
);
Если вы передадите только одного ребёнка, ничего страшного, эта функция поймёт, что работает не с массивом детей, и выведет ребёнка, как будто пришел массив с одним элементом.
